BasePoint POS (Duka) supports thermal receipt printing out of the box, with full control over how your receipts look. You can set the document type, toggle VAT and discount lines on or off, adjust font size and weight, and choose between an 80mm thermal roll format or a full A4 PDF layout. After every confirmed payment, the Print Bill modal gives your cashier a one-tap path to print, download, or email the receipt to the customer.

## Supported Printers

BasePoint POS is compatible with standard **80mm thermal receipt printers** that use the ESC/POS command set — the industry-standard protocol supported by most popular thermal printer brands. Printing uses your operating system's native print dialog via the browser, so no proprietary software is required beyond the printer driver.

<Info>
  On Linux-based POS terminals, thermal printing requires **CUPS** (Common Unix
  Printing System) and a compatible PPD driver file. Your BasePoint reseller
  provides the PPD file for supported printer models.
</Info>

## Setting Up Your Printer

<Note>
  The steps below (CUPS installation and driver setup) are system-level
  configuration tasks typically performed by your **IT team** or system
  administrator before handing the terminal over to cashiers. Regular POS
  users do not need to complete these steps.
</Note>

<Steps>
  <Step title="Install CUPS (Linux only)">
    Open a terminal and run:

    ```bash
    sudo apt install cups
    ```

    On Windows or macOS, install the printer driver supplied by your hardware vendor and add the printer through the operating system's standard printer settings.
  </Step>
  <Step title="Install the thermal printer driver">
    Obtain the **PPD file** for your printer model from your BasePoint reseller or hardware supplier. On Linux, add the printer in CUPS at `http://localhost:631` and select the PPD file when prompted. On macOS, add the printer in **System Settings** → **Printers & Scanners** and choose the PPD file.
  </Step>
  <Step title="Test print from the OS">
    Print a test page from the OS printer settings to confirm the driver is working and the paper feeds correctly before configuring BasePoint POS.
  </Step>
  <Step title="Select the printer in BasePoint POS">
    In BasePoint POS, navigate to **System Setup** → **Printer Settings**. Select your newly added printer from the list of available printers.
  </Step>
  <Step title="Send a test print">
    Click **Test Print** to send a sample receipt to the printer. Confirm the output is correctly formatted on the 80mm roll before going live.
  </Step>
</Steps>

<Note>
  BasePoint POS uses browser-based printing for thermal output. Use **Google
  Chrome** or **Brave** for the most reliable results. Other browsers may
  override the 80mm page size or add unwanted headers and footers to the
  printed output.
</Note>

## Printing a Receipt

After you confirm a payment, the **Print Bill** modal appears automatically. It previews the receipt on screen so you can verify it before printing. From this modal you can customize the output and choose how to deliver it:

### Document Type

Select the type of document to print using the **Document Type** selector:

- **Bill** — an itemized pre-payment or in-progress bill.
- **Receipt** — a post-payment confirmation document.
- **Invoice** — a formal invoice for business clients.
- **Quotation** — a price estimate valid for 30 days.

### Print Format

Toggle between **Thermal (POS)** and **A4 PDF** mode using the format switch:

- **Thermal mode** — sends output to the configured 80mm thermal printer via the browser print dialog.
- **A4 PDF mode** — renders a professional A4 layout. Use the **Save as PDF** action to save it locally.

### Receipt Appearance

When in thermal mode, use the controls in the Print Bill modal to adjust how the receipt looks before printing:

<CardGroup cols={2}>
  <Card title="Font Size" icon="text-size" href="#receipt-appearance">
    Choose a font size preset — **Small**, **Normal**, **Large**, **X-Large**, **XX-Large**, or **Huge** — or drag the slider to a custom size. Font size is clamped on the printed 80mm output to prevent line overflow.
  </Card>
  <Card title="Text Weight" icon="bold" href="#receipt-appearance">
    Toggle between **Bold** and **Normal** text weight. Bold improves contrast and is easier to scan under low-light conditions.
  </Card>
  <Card title="Show Discount" icon="percent" href="#receipt-appearance">
    Toggle the discount line on or off. Only active when a discount has been applied to the order.
  </Card>
  <Card title="Show VAT" icon="file-invoice" href="#receipt-appearance">
    Toggle the VAT breakdown line on or off on the printed document.
  </Card>
</CardGroup>

### Delivery Options

From the Print Bill modal footer, use any of the following actions:

- **Print Document** — sends the receipt to the configured thermal printer in 80mm format.
- **Save as PDF** — available when A4 PDF mode is active; saves a PDF copy to your device.
- **Send via Email** — opens an email form. Enter the customer's email address and an optional personal message; the receipt is delivered as a formatted email with the order summary.
- **Download PDF** — saves a local PDF copy directly to your device, regardless of the current format mode.
- **Cancel** — closes the modal without printing. You can reprint any order later from the **Orders** list.

<Tip>
  Use the **Print Bill** button on an open, unpaid cart to show the customer
  a pre-payment bill total before they hand over cash or a card. This is
  different from a receipt — it prints the current cart as a "Bill" document
  rather than a confirmed "Receipt."
</Tip>

## Header and Business Information

Your shop name, address, phone number, PIN, and email are pulled automatically from your business profile and printed at the top of every receipt. To update this information, go to **Settings** → **Business Profile**.

If **DigiTax / ETR** integration is enabled, a KRA verification QR code and ETR receipt number are printed automatically on every confirmed transaction — no extra steps required.
